# راهنمای مشارکت در پروژه GraphRAG

## مقدمه

از مشارکت شما در پروژه GraphRAG استقبال می‌کنیم! این سند راهنمای مشارکت در پروژه است.

## نحوه مشارکت

### 1. Fork کردن پروژه
- پروژه را در GitHub fork کنید
- پروژه را به کامپیوتر خود clone کنید

### 2. ایجاد Branch جدید
```bash
git checkout -b feature/your-feature-name
```

### 3. اعمال تغییرات
- کد خود را بنویسید
- تست‌های مناسب اضافه کنید
- مستندات را به‌روزرسانی کنید

### 4. Commit کردن تغییرات
```bash
git add .
git commit -m "Add: توضیح کوتاه تغییرات"
```

### 5. Push کردن تغییرات
```bash
git push origin feature/your-feature-name
```

### 6. ایجاد Pull Request
- به repository اصلی بروید
- Pull Request جدید ایجاد کنید
- توضیحات کامل ارائه دهید

## استانداردهای کدنویسی

### Python
- از PEP 8 پیروی کنید
- از type hints استفاده کنید
- docstring برای توابع بنویسید
- نام‌گذاری متغیرها به انگلیسی باشد

### Commit Messages
- از فرمت زیر استفاده کنید:
  - `Add:` برای اضافه کردن ویژگی جدید
  - `Fix:` برای رفع باگ
  - `Update:` برای به‌روزرسانی
  - `Remove:` برای حذف
  - `Refactor:` برای بازنویسی

### مستندات
- README.md را به‌روزرسانی کنید
- تغییرات API را مستند کنید
- مثال‌های استفاده اضافه کنید

## تست‌ها

### اجرای تست‌ها
```bash
python -m pytest tests/
```

### پوشش تست
```bash
python -m pytest --cov=graphrag_service tests/
```

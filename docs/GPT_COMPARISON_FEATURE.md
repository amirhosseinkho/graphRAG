# ویژگی مقایسه هوشمند با GPT-4o

## خلاصه
این ویژگی جدید امکان مقایسه هوشمند دو متن را با استفاده از GPT-4o فراهم می‌کند. کاربران می‌توانند دو پاسخ مختلف را که با روش‌های متفاوت تولید شده‌اند، مقایسه کنند و تحلیل دقیقی از کیفیت هر کدام دریافت کنند.

## نحوه استفاده

### 1. دسترسی به صفحه ارزیابی
- به صفحه ارزیابی (`/evaluation`) بروید
- بخش جدید "مقایسه هوشمند با GPT-4o" را مشاهده خواهید کرد

### 2. تنظیم پارامترها
- **برچسب روش اول**: نام روش اول (مثل "بازیابی از گراف")
- **برچسب روش دوم**: نام روش دوم (مثل "تولید مستقیم")
- **نوع مقایسه**: انتخاب نوع تحلیل
  - مقایسه جامع (کیفیت، دقت، جامعیت)
  - تمرکز بر دقت و صحت اطلاعات
  - تمرکز بر جامعیت و کامل بودن
  - تمرکز بر وضوح و قابل فهم بودن
  - تمرکز بر مرتبط بودن با سوال

### 3. وارد کردن متن‌ها
- دو متن مختلف را در بخش‌های پاسخ اول و دوم وارد کنید
- یا از دکمه "تولید خودکار پاسخ‌ها" استفاده کنید

### 4. اجرای مقایسه
- روی دکمه "مقایسه با GPT-4o" کلیک کنید
- منتظر بمانید تا تحلیل تکمیل شود

## خروجی تحلیل

### خلاصه مقایسه
تحلیل کلی از تفاوت‌های اصلی بین دو متن

### امتیازدهی
- امتیاز از 1 تا 10 برای هر روش
- توضیح دلیل امتیازدهی

### تحلیل جزئیات
- **نقاط قوت**: ویژگی‌های مثبت هر روش
- **نقاط ضعف**: محدودیت‌های هر روش

### توصیه نهایی
پیشنهاد بهترین روش بر اساس تحلیل انجام شده

## ویژگی‌های فنی

### API Endpoint
```
POST /api/compare_with_gpt
```

### پارامترهای ورودی
```json
{
    "text1": "متن اول",
    "text2": "متن دوم", 
    "label1": "برچسب روش اول",
    "label2": "برچسب روش دوم",
    "comparison_type": "نوع مقایسه"
}
```

### خروجی
```json
{
    "summary": "خلاصه مقایسه",
    "score1": 8,
    "score2": 6,
    "scoring_explanation": "توضیح امتیازدهی",
    "strengths1": "نقاط قوت روش اول",
    "strengths2": "نقاط قوت روش دوم", 
    "weaknesses1": "نقاط ضعف روش اول",
    "weaknesses2": "نقاط ضعف روش دوم",
    "recommendation": "توصیه نهایی",
    "label1": "برچسب روش اول",
    "label2": "برچسب روش دوم",
    "comparison_type": "نوع مقایسه"
}
```

## نیازمندی‌ها

### کتابخانه‌های مورد نیاز
```bash
pip install openai
```

### تنظیمات OpenAI
- API Key معتبر OpenAI نیاز است
- مدل GPT-4o برای بهترین نتایج

## نمونه استفاده

### مثال 1: مقایسه روش‌های بازیابی
```javascript
// ارسال درخواست
fetch('/api/compare_with_gpt', {
    method: 'POST',
    headers: {'Content-Type': 'application/json'},
    body: JSON.stringify({
        text1: "پاسخ با روش BFS",
        text2: "پاسخ با روش DFS", 
        label1: "BFS",
        label2: "DFS",
        comparison_type: "comprehensive"
    })
})
```

### مثال 2: مقایسه مدل‌های مختلف
```javascript
// مقایسه مدل‌های تولید متن
{
    text1: "پاسخ HuggingFace",
    text2: "پاسخ OpenAI GPT",
    label1: "HuggingFace",
    label2: "OpenAI GPT", 
    comparison_type: "accuracy"
}
```

## نکات مهم

### محدودیت‌ها
- نیاز به اتصال اینترنت برای دسترسی به GPT-4o
- هزینه API calls بر اساس استفاده
- محدودیت طول متن ورودی

### بهترین شیوه‌ها
- متن‌های با کیفیت و کامل وارد کنید
- برچسب‌های توصیفی و واضح استفاده کنید
- نوع مقایسه مناسب با نیاز خود انتخاب کنید

### عیب‌یابی
- در صورت خطا در اتصال، بررسی کنید که OpenAI API Key معتبر باشد
- برای خطاهای طولانی، بررسی کنید که متن‌ها خیلی طولانی نباشند
- در صورت عدم دسترسی به GPT-4o، از روش‌های مقایسه محلی استفاده کنید

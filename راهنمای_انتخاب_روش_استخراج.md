# راهنمای انتخاب روش استخراج

این راهنما به شما کمک می‌کند بهترین روش استخراج را برای متن خود انتخاب کنید.

## روش‌های استخراج

### 1. Simple (Rule-based)
**زمان:** سریع  
**دقت:** متوسط  
**مناسب برای:**
- متن‌های ساختاریافته
- متن‌های کوتاه
- موجودیت‌های واضح (مثلاً نام ژن‌ها با حروف بزرگ)

**مثال:**
```
TP53 regulates apoptosis. BRCA1 interacts with TP53.
```

---

### 2. spaCy
**زمان:** متوسط  
**دقت:** خوب  
**مناسب برای:**
- متن‌های انگلیسی
- متن‌های علمی و تخصصی
- نیاز به استخراج دقیق موجودیت‌ها

**نکته:** برای فارسی، از روش "Persian" استفاده کنید.

---

### 3. spaCy SVO Enhanced
**زمان:** متوسط  
**دقت:** خوب  
**مناسب برای:**
- استخراج روابط Subject-Verb-Object
- متن‌هایی که روابط واضح دارند
- تحلیل ساختار جمله

---

### 4. LLM (Single-pass)
**زمان:** کند  
**دقت:** عالی  
**مناسب برای:**
- متن‌های پیچیده
- نیاز به دقت بالا
- متن‌های طولانی (با chunking)

**نیاز:** توکن HuggingFace

---

### 5. LLM Multi-pass
**زمان:** خیلی کند  
**دقت:** بسیار عالی  
**مناسب برای:**
- حداکثر recall (یافتن همه موجودیت‌ها)
- متن‌های مهم که نیاز به دقت کامل دارند
- استخراج تکراری برای اطمینان

**نیاز:** توکن HuggingFace

---

### 6. Hybrid
**زمان:** متوسط تا کند  
**دقت:** عالی  
**مناسب برای:**
- ترکیب نتایج چندین روش
- بهترین balance بین دقت و سرعت
- متن‌های مهم

---

### 7. Persian
**زمان:** متوسط  
**دقت:** خوب  
**مناسب برای:**
- متن‌های فارسی
- استفاده از مدل‌های ParsBERT و mT5
- پردازش زبان فارسی

---

### 8. Span-based
**زمان:** متوسط  
**دقت:** عالی  
**مناسب برای:**
- موجودیت‌های زیست‌پزشکی (BioBERT)
- متن‌های علمی (SciBERT)
- نیاز به دقت بالا در شناسایی موجودیت‌ها

---

### 9. With Coreference
**زمان:** کند  
**دقت:** عالی  
**مناسب برای:**
- متن‌هایی با ارجاعات زیاد ("این ژن"، "آن پروتئین")
- ادغام ارجاعات با موجودیت‌های اصلی
- بهبود کیفیت گراف

---

### 10. Long Text
**زمان:** کند  
**دقت:** خوب تا عالی  
**مناسب برای:**
- متن‌های بسیار طولانی
- مقالات و کتاب‌ها
- پردازش با chunking هوشمند

---

## راهنمای سریع انتخاب

| نوع متن | روش پیشنهادی |
|--------|-------------|
| متن کوتاه و ساده | Simple |
| متن انگلیسی علمی | spaCy یا Span-based |
| متن فارسی | Persian |
| متن طولانی | Long Text |
| نیاز به دقت بالا | LLM یا Hybrid |
| حداکثر recall | LLM Multi-pass |
| ویکی‌پدیا | Persian یا spaCy (با استخراج تخصصی) |
| متن با ارجاعات | With Coreference |

---

## برای ویکی‌پدیا

### استخراج تخصصی ویکی‌پدیا

وقتی URL ویکی‌پدیا را وارد می‌کنید، سیستم به صورت خودکار:

1. **Infobox را استخراج می‌کند** - اطلاعات ساختاریافته
2. **Categories را استخراج می‌کند** - دسته‌بندی‌ها
3. **لینک‌های داخلی را استخراج می‌کند** - روابط با صفحات دیگر
4. **بخش‌ها را استخراج می‌کند** - ساختار صفحه

### توصیه برای ویکی‌پدیا:

- **روش استخراج:** `Persian` (برای فارسی) یا `spacy` (برای انگلیسی)
- **استخراج تخصصی:** فعال باشد (پیش‌فرض)
- **Coreference Resolution:** برای صفحات پیچیده فعال کنید

---

## نکات مهم

1. **برای URL ویکی‌پدیا:** همیشه استخراج تخصصی را فعال کنید
2. **برای متن‌های طولانی:** از `Long Text` با `chunking_strategy: smart` استفاده کنید
3. **برای دقت بالا:** از `LLM` یا `Hybrid` استفاده کنید
4. **برای سرعت:** از `Simple` یا `spaCy` استفاده کنید

---

## مثال استفاده

### ویکی‌پدیا فارسی:
```
URL: https://fa.wikipedia.org/wiki/سرطان
Method: Persian
استخراج تخصصی: فعال
```

### متن علمی انگلیسی:
```
Text: "TP53 regulates apoptosis..."
Method: Span-based (BioBERT)
```

### متن طولانی:
```
Text: [مقاله طولانی]
Method: Long Text
Chunking: Smart
Overlap: 0.2
```
